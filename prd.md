# PRD: scopy — 채용 시장 탐색 대시보드 (v2.4)

> v1(기업 채용담당자용 ATS 대시보드)은 폐기하고, 구직자 관점으로 전면 재기획함.
> v2.1: 원티드 실 API 연동 범위를 넓히고(공고 탐색·북마크·자소서 추천), 자소서 관리·기업 추천 기능을 추가함.
> v2.2: 시장 개요도 실 API로 전환 — 직군별 공고는 전수 집계(offset 이진탐색), 지역별 분포·기업 매력태그·마감 임박 공고 차트 신설. 등록일시가 API에 없어 신규 공고 추이·기간 필터는 제거함.
> v2.3: 기업 비교에 실 데이터 기반 레이더(육각형) 비교 추가. 자소서 관리·최근 방문 공고를 마이페이지로 통합하고 자격증·선호 직군을 신설 — 선호 직군은 공고 탐색 필터와 연동. 이 확장에는 신규 원티드 API 호출이 필요 없음(§3-1 참고).
> v2.4: 마지막까지 가상 데이터로 남아있던 기업 비교(표·차트·레이더)를 LIVE 표본의 기업 단위 집계로 전면 교체 — 평균연봉·입사율·퇴사율·1인당매출·인원은 `/v1/insight/company`가 여전히 401이라 못 내지만, 가짜 값 대신 그 항목 자체를 화면에서 뺐다. 자소서 키워드 사전(ALL_KEYWORDS)에 섞이던 가상 태그 카탈로그(D.tags)도 제거하고 LIVE 표본 키워드만 사용. `db/generate_seed.py` 가상 시드(tags/companies/jobs)는 `js/data.js`에서 완전히 제거해 전 화면이 원티드 실 API 데이터만 사용함.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 제품명 | scopy (Job Market Explorer) |
| 한 줄 정의 | 원티드(Wanted) 공개 API로 채용 시장을 탐색하고, 내 자소서에 맞는 기업까지 찾아보는 구직자용 대시보드 |
| 타깃 사용자 | 이직·구직 중인 지식노동자 (특히 개발 직군) |
| 핵심 가치 | 실제 지원 가능한 공고 탐색 + 스킬 수요 파악 + 기업별 채용 활발도·보상금·복지 비교 + 자소서 기반 기업 추천을 한 화면에서 |
| 데이터 소스 | Wanted OpenAPI V2 (`/jobs`) · V1 (`/jobs/{id}`, `/search/*`, `/companies/*`, `/insight/company`, `/tags/*`) |

## 2. 문제 정의

구직자는 "어떤 공고가 있는지"는 채용 플랫폼에서 볼 수 있지만,
**시장 전체의 수요(직군·스킬별 공고량)**와 **기업별 채용 활발도·보상금·복지 수준**을
공고와 나란히 놓고 비교할 수 있는 화면이 없다. 또한 자소서를 써놓고도 "이 글이 어떤 회사·직무에
맞는지"를 스스로 판단하기 어렵다. 원티드 API는 앞의 두 가지를 제공하고, 세 번째는 API가 반환하는
직군·스킬·매력태그 데이터를 자소서 텍스트와 매칭해 보완할 수 있다. (평균연봉·퇴사율 같은 재무·인사
지표는 `/v1/insight/company`가 401 권한 없음이라 이 앱에서는 다루지 않는다.)

## 3. 참조 API 매핑 — 연동 현황

파트너 키로 실제 호출해 확인한 결과, 엔드포인트별로 접근 가능 범위가 다르다. **연동** = 지금 실제로
호출해서 화면에 반영 중. **가상 데이터로 대체** = API가 필요한 정보를 안 주거나(등록일시) 권한이
없어서(401) 결정적 시드 데이터로 대신 채움. **제외** = 유료 계약 또는 기업 전용이라 이 앱 범위 밖.

| API | 화면·기능 | 상태 |
|---|---|---|
| `GET /v2/jobs` | 공고 탐색 목록 (직군·경력·정렬 필터) + 시장 개요 직군별 전수 공고 수 (limit=1 offset 이진탐색) | ✅ 연동 — `db/fetch_live_jobs.py` |
| `GET /v1/jobs/{job_id}` | 공고 상세 보강 (스킬 태그, 경력, 마감일, 기업의 매력태그) — 시장 개요 표본 차트의 원천 | ✅ 연동 — 위와 동일 스크립트 |
| `GET /v1/tags/categories` | 직군 태그 카탈로그, 자소서 키워드 사전 | ✅ 연동 |
| `GET /v1/tags/attractions` | 매력태그(복지·문화) 카탈로그 — 실 공고엔 `jobs/{id}`의 `company.company_tags`로 이미 포함되어 별도 호출 불필요 | ✅ 연동 |
| `GET /v1/insight/company` | 기업 인사이트 (평균연봉·입사/퇴사율·매출) | ❌ 401 권한 없음 — **해당 지표는 화면에서 제외**(가상 데이터로 대체하지 않음). 기업 비교는 LIVE를 기업 단위로 집계한 다른 지표(진행중 공고·추천보상금·복지태그·요구스킬·최소경력·지원여유)로 대체 |
| `GET /v1/search/position`, `/search/company`, `/search/company/autocomplete` | 공고 탐색 검색창 자동완성 | 🔜 미연동 — §3-1 확장 로드맵 |
| `GET /v1/companies/{id}/jobs` | "이 기업의 다른 진행중 공고" 딥링크 | 🔜 미연동 — §3-1 확장 로드맵 |
| `GET /v1/tags/skills` | 자소서 키워드 사전 최신화 (스킬 카탈로그 동기화) | 🔜 미연동 — §3-1 확장 로드맵 |
| `POST /ai/pass·apply/text-prediction/async` | 자소서 합격/지원 예측 | 🚫 제외 — 별도 계약 전용(`team-ml@wantedlab.com`), 405/계약 필요 |
| `/ats/*`, `/recruit-company/*`, `/stat/*` | 공고 등록·지원서 관리 | 🚫 제외 — 기업(구인 측) 전용, 401 |

인증: 모든 호출에 `wanted-client-id` / `wanted-client-secret` 헤더. 두 값은 `.env`에만 존재하며
(`WANTED_CLIENT_ID`, `WANTED_CLIENT_SECRET`) git에는 절대 포함하지 않는다 — 실 API 호출은
빌드 타임 스크립트(`db/fetch_live_jobs.py`)에서만 수행하고, 그 결과(공개 채용 정보)만 `js/data.js`에
정적으로 굽는다. 브라우저 런타임에서 시크릿을 직접 호출하지 않는다.

### 3-1. API 활용 확장 로드맵 (현 모델 유지, 범위만 확대)

새 화면·새 비즈니스 모델을 만들지 않고, 지금 있는 화면 안에서 실 API 호출을 더 늘리는 항목만 정리.

| 항목 | 사용 API | 붙는 위치 | 우선순위 |
|---|---|---|---|
| 검색 자동완성 | `GET /v1/search/position`, `/v1/search/company/autocomplete` | 공고 탐색 검색창 — 입력 중 포지션명·기업명 후보 제시 | P1 |
| 기업의 다른 공고 보기 | `GET /v1/companies/{id}/jobs` | 공고 카드·자소서 추천 결과에서 기업명 클릭 시 펼쳐보기 | P1 |
| 스킬 사전 동기화 | `GET /v1/tags/skills?keyword=` | `fetch_live_jobs.py` 실행 시 자소서 키워드 사전을 실 API 스킬 카탈로그로 보강 | P2 |

## 5. 화면 구성

1. **시장 개요** — KPI 4종 + 직군별 전수 공고 + 스킬 Top 10 + 경력대 분포 + 지역별 분포 + 기업 매력태그 Top 10 + 마감 임박 공고 (실 API, 수집 시각 표시)
2. **공고 탐색** — 필터 툴바(선호 직군이 있으면 기본 선택됨) + 공고 카드 그리드, 카드 클릭 시 실제 지원 페이지로 이동 (실 API)
3. **기업 비교** — 추천보상금·채용 활발 Top 10 차트 + 기업 인사이트 테이블(행 클릭 시 A/B 레이더 선택) + 기업 2곳 육각형 레이더 나란히 비교 (전부 실 API 표본 집계)
4. **북마크** — 저장한 공고 모아보기 (실 API)
5. **마이페이지** — 자격증 + 선호 직군 + 자소서 작성/저장/추천 기업 + 최근 방문 공고 (자격증·선호 직군은 로컬, 나머지는 실 API)

## 6. 디자인 원칙 (v1 유지)

- 모던 & 트렌디: 웜 뉴트럴 서페이스, 카드 레이아웃, 넉넉한 여백. **네온 스타일 절대 금지.**
- 검증된 저채도 데이터 팔레트, 얇은 마크, 헤어라인 그리드, 색은 항목에 고정.
- Pretendard + 시스템 산세리프 폴백. 라이트/다크 모두 지원.
- 화면에 실 데이터와 가상 데이터가 섞이는 경우가 생기면 반드시 card-sub 문구로 데이터 성격을 밝힌다 — 사용자가 진짜 지표와 가상 지표를 혼동하지 않도록. (v2.4 기준 전 화면이 실 API라 현재 해당 사항 없음 — 향후 새 화면에 적용할 원칙으로 유지)

## 7. 기술 구성

정적 SPA(HTML/CSS/Vanilla JS, 브라우저에서 바로 실행)를 유지한다. v2.4부터는 모든 화면이
`liveJobs` 하나만 쓴다 — 가상 데이터 이원화는 폐기.

| 레이어 | 선택 |
|---|---|
| 프론트엔드 | 정적 SPA (HTML/CSS/Vanilla JS) — 빌드 불필요 |
| 차트 | 자체 SVG 렌더링 (`js/charts.js`) |
| 데이터 | 원티드 OpenAPI를 빌드 타임에 호출하는 `db/fetch_live_jobs.py` — 전 화면이 이 결과(`liveJobs`)를 씀. API 시크릿은 `.env`(git 제외)에만 존재, 브라우저에는 노출되지 않음 |
| 데이터 내보내기 | `db/export_data.sh`가 `fetch_live_jobs.py` 실행 결과만으로 `js/data.js` 생성 (`liveJobs`/`liveCategoryTotals`/`liveFetchedAt`). API 호출 실패 시 직전 `js/data.js`의 liveJobs를 그대로 유지, 그마저 없으면 빈 배열로 진행 |
| 레거시(미사용) | `db/scopy.db`·`db/generate_seed.py`·`schema.sql`·`seed.sql` — v2.3까지 기업 비교가 쓰던 가상 시드. v2.4부터 `export_data.sh`가 더 이상 참조하지 않음. 리포에는 남아있지만 삭제하지 않은 것뿐, 실행 경로에서 완전히 빠짐 |

## 8. DB 설계 요약 (레거시 — v2.4부터 미사용)

아래 스키마(`db/scopy.db`)는 v2.3까지 기업 비교가 쓰던 가상 시드의 흔적이다. v2.4에서 기업
비교를 LIVE 집계로 교체하면서 참조가 끊겼다 — 삭제하진 않았지만 `js/data.js`에도 더 이상
포함되지 않고, 코드 어디에서도 읽지 않는다.

- `companies` — 기업 + 가상 `/insight/company` 지표 (평균연봉, 입사/퇴사율, 인원, 매출, 업력)
- `tags` — 직군/직무/스킬/매력 태그 (`tag_type`, 직무는 `parent_id`로 직군 참조)
- `company_tags` — 기업-매력태그 N:M (`/v1/tags/attractions` 카탈로그 기준, 기업 규모급별 가중 배정)
- `jobs` — 공개 채용공고 (V2 `/jobs` + V1 `/jobs/{id}` 응답 필드: 상태, 경력, 고용형태, 보상금, 지역, 등록일, 마감일, URL)
- `job_tags` — 공고-태그 N:M (직무/스킬)
- `bookmarks` — 관심 공고 (클라이언트는 localStorage 사용, 서버 전환 대비 테이블 정의)

## 9. 제외 범위 (Out of Scope)

- 로그인/회원 (북마크·자소서는 로컬 저장)
- 지원하기 액션 (원티드 공고 URL로 이동만)
- 자소서 합격/지원 예측 AI (`/ai/pass·apply/text-prediction/async`) — 별도 유료 계약 전용, 이 키로 호출 불가. 대체로 직군·스킬·매력태그 키워드 매칭 기반 기업 추천(F-13)을 구현
- 기업 인사이트 실 연동 (`/insight/company`) — 401 권한 없음. 평균연봉·입사율·퇴사율·1인당매출·인원은 가상 값으로 채우지 않고 화면에서 아예 제외 (v2.4)
- 기업 전용 API (`/ats/*`, `/recruit-company/*`, `/stat/*`) — 구인 기업 전용, 이 앱의 구직자 타깃과 무관
- 알림/구독

## 10. 성공 기준

- 진입 5초 안에 "내 직군에서 지금 어떤 스킬이 수요가 많은지" 파악 가능
- 시장 개요의 직군별 공고 수가 원티드 실제 총량과 일치 (전수 집계), 수집 시각이 화면에 표시됨
- 공고 탐색에서 카드를 클릭하면 실제로 지원 가능한(마감되지 않았거나, 마감이면 실제 마감으로 표시되는) 원티드 공고로 이동
- 화면에 보이는 모든 수치가 원티드 실 API 표본에서 나온 값 — 가상/가짜 지표가 섞여있지 않음 (v2.4)
- 자소서를 쓰면 30자 이상 입력 시점부터 관련 키워드와 추천 기업이 뜨고, 클릭 시 해당 기업 홈페이지로 이동
