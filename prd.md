# PRD: scopy — 채용 시장 탐색 대시보드 (v2)

> v1(기업 채용담당자용 ATS 대시보드)은 폐기하고, 구직자 관점으로 전면 재기획함.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 제품명 | scopy (Job Market Explorer) |
| 한 줄 정의 | 원티드(Wanted) 공개 API로 채용 시장을 탐색하고 기업을 비교하는 구직자용 대시보드 |
| 타깃 사용자 | 이직·구직 중인 지식노동자 (특히 개발 직군) |
| 핵심 가치 | 공고 탐색 + 스킬 수요 파악 + 기업 건전성(연봉·퇴사율) 비교를 한 화면에서 |
| 데이터 소스 | Wanted OpenAPI V2 (`/jobs`) · V1 (`/jobs/{id}`, `/search/*`, `/companies/*`, `/insight/company`, `/tags/*`) |

## 2. 문제 정의

구직자는 "어떤 공고가 있는지"는 채용 플랫폼에서 볼 수 있지만,
**시장 전체의 수요(직군·스킬별 공고량)**와 **회사의 실제 상태(평균연봉, 퇴사율, 인원 변동)**를
공고와 나란히 놓고 비교할 수 있는 화면이 없다. 원티드 API는 이 세 가지를 모두 제공한다.

## 3. 참조 API 매핑

| 대시보드 기능 | 원티드 API |
|---|---|
| 공고 탐색 (직군·스킬·경력·정렬 필터) | `GET /v2/jobs` |
| 공고 상세 (보상금·주소·태그) | `GET /v1/jobs/{job_id}` |
| 포지션·기업 검색 | `GET /v1/search/position`, `/search/company` |
| 기업 정보·기업의 다른 공고 | `GET /v1/companies/{id}`, `/companies/{id}/jobs` |
| 기업 인사이트 (평균연봉·입사/퇴사율·인원·매출) | `GET /v1/insight/company` |
| 직군/직무/스킬 태그 축 | `GET /v1/tags/categories`, `/tags/skills` |

인증: 모든 호출에 `wanted-client-id` / `wanted-client-secret` 헤더.

## 4. 핵심 기능

| No | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F-1 | 시장 KPI | 신규 공고 수·채용 기업 수·평균 보상금·신입 가능 공고 (+이전 기간 대비) | P0 |
| F-2 | 신규 공고 추이 | 기간별 신규 등록 공고 추이 라인 차트 | P0 |
| F-3 | 직군별 공고 분포 | 직군 × 공고 수 비교 | P0 |
| F-4 | 스킬 수요 Top 10 | 공고에 붙은 스킬 태그 빈도 상위 10개 | P0 |
| F-5 | 경력대별 공고 분포 | 신입/주니어/미들/시니어 구간별 공고 수 | P1 |
| F-6 | 공고 탐색 | 카드형 리스트, 검색 + 직군·경력·고용형태 필터, 정렬(최신/보상금) | P0 |
| F-7 | 기업 비교 | 평균연봉·퇴사율·인원·진행중 공고 수 테이블 + 연봉 Top 10 차트 | P0 |
| F-8 | 북마크 | 관심 공고 저장(로컬), 모아보기 | P1 |
| F-9 | 기간 필터 | 최근 7/30/90일/전체 — 시장 개요의 모든 카드·차트에 일괄 적용 | P0 |
| F-10 | 표 뷰·다크 모드 | 모든 차트에 표 대체 뷰, 라이트/다크 테마 | P1 |

## 5. 화면 구성

1. **시장 개요** — KPI 4종 + 신규 공고 추이 + 직군별 분포 + 스킬 Top 10 + 경력대 분포
2. **공고 탐색** — 필터 툴바 + 공고 카드 그리드 (기업명·직군·스킬·경력·보상금·마감 D-day·북마크)
3. **기업 비교** — 기업 테이블(공고 수 순) + 평균연봉 Top 10 차트
4. **북마크** — 저장한 공고 모아보기

## 6. 디자인 원칙 (v1 유지)

- 모던 & 트렌디: 웜 뉴트럴 서페이스, 카드 레이아웃, 넉넉한 여백. **네온 스타일 절대 금지.**
- 검증된 저채도 데이터 팔레트, 얇은 마크, 헤어라인 그리드, 색은 항목에 고정.
- Pretendard + 시스템 산세리프 폴백. 라이트/다크 모두 지원.

## 7. 기술 구성 (v1 유지)

| 레이어 | 선택 |
|---|---|
| 프론트엔드 | 정적 SPA (HTML/CSS/Vanilla JS) — 브라우저에서 바로 실행 |
| 차트 | 자체 SVG 렌더링 (`js/charts.js`) |
| DB | SQLite (`db/scopy.db`) — 스키마 `db/schema.sql`, 시드 `db/seed.sql` |
| 데이터 주입 | SQLite → `js/data.js` (`db/export_data.sh`) — 실서비스 전환 시 이 로더만 API 호출로 교체 |

## 8. DB 설계 요약

- `companies` — 기업 + `/insight/company` 지표 (평균연봉, 입사/퇴사율, 인원, 매출, 업력)
- `tags` — 직군/직무/스킬 태그 (`tag_type`, 직무는 `parent_id`로 직군 참조)
- `jobs` — 공개 채용공고 (V2 `/jobs` + V1 `/jobs/{id}` 응답 필드: 상태, 경력, 고용형태, 보상금, 지역, 등록일, 마감일, URL)
- `job_tags` — 공고-태그 N:M (직무/스킬)
- `bookmarks` — 관심 공고 (클라이언트는 localStorage 사용, 서버 전환 대비 테이블 정의)

## 9. 제외 범위 (Out of Scope)

- 로그인/회원 (북마크는 로컬 저장)
- 지원하기 액션 (원티드 공고 URL로 이동만)
- 기업 전용 API (`/ats/*`, `/recruit-company/*`, `/stat/*`, `/ai/*`)
- 알림/구독

## 10. 성공 기준

- 진입 5초 안에 "내 직군에서 지금 어떤 스킬이 수요가 많은지" 파악 가능
- 공고 카드에서 기업 인사이트(연봉·퇴사율)까지 클릭 한 번 이내 도달
- 기간 필터 변경 시 시장 개요 전체가 동일 슬라이스로 즉시 갱신
