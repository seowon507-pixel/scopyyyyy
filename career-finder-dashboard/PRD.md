# PRD — 커리어 파인더 대시보드 (Career Finder Dashboard)

> 작성: 메인 에이전트 · 2026.07.13
> 근거 자료: `career-dashboard.html`(디자인 시안), `openapi.json`(Wanted OpenAPI v1), `openapi-2.json`(v2 jobs)

## 1. 한 줄 정의

구직자가 자신의 지원 현황과 목표 준비도를 관리하면서, 원티드(Wanted) 공개 API로 관심기업·공고·채용시장 데이터를 **실시간으로 동기화**해 보여주는 개인용 구직 활동 대시보드.

## 2. 대상 사용자 (Persona)

- **주 타깃**: 데이터/서비스기획 등 실무직군에 지원 중인 취업준비생·이직 준비자 (신입~경력 3년 이하)
- **특징**: 여러 회사에 동시 지원 중이라 전형 단계가 뒤섞여 관리가 안 됨 / 관심기업의 채용 상태(공고 마감·신규 오픈)를 수동으로 매번 확인하는 데 지쳐 있음 / 스킬·자격증 준비 진행률을 한눈에 보고 싶어 함
- 1인 사용 기준 (팀/조직 대시보드 아님)

## 3. 핵심 가치 (Value Proposition)

1. **한 화면에서 지원 파이프라인 전체 파악** — 지원→서류→면접→합격 퍼널과 다가오는 일정을 동시에 확인
2. **실시간 동기화** — 관심기업의 공고 상태(모집중/마감임박/상시채용), 신규 공고, 시장 트렌드가 자동으로 갱신되어 새로고침 없이 항상 최신 상태를 봄
3. **데이터 기반 준비 전략** — 원티드 공개 데이터(직군별 공고량, 인기 스킬 태그)를 근거로 내 스킬 준비도와 비교

## 4. 필수 기능 (Must-have)

| # | 기능 | 설명 | 데이터 출처 |
|---|------|------|------|
| 1 | **실시간 동기화 (Realtime Sync)** | Supabase Realtime 구독으로 지원현황·일정·관심기업 상태 변경이 다른 탭/기기/백그라운드 갱신에서 즉시 화면에 반영됨. 폴링이 아닌 이벤트 기반 push. | Supabase Postgres Changes |
| 2 | 지원 현황 파이프라인 & 목록 | 지원완료→서류전형→면접→최종합격 단계별 건수, 전환율, 필터 가능한 지원 내역 테이블 (CRUD) | 사용자 입력 (자체 DB) |
| 3 | 일정 관리 | 면접/과제마감/결과발표 일정과 D-day, 임박도에 따른 색상 강조 | 사용자 입력 (자체 DB) |
| 4 | 관심기업 워치리스트 + 실시간 채용상태 | 관심 등록한 기업의 최신 채용중 포지션 수·마감 임박 여부를 Wanted API로 주기 동기화 | `GET /companies/{id}`, `GET /companies/{id}/jobs` |
| 5 | 공고/기업 검색 & 추가 | 이름으로 기업·포지션 검색 후 워치리스트/지원목록에 바로 추가 | `GET /search/company`, `GET /search/position`, `GET /jobs` |
| 6 | 채용시장 인사이트 | 직군(카테고리)별 공고 수 추이, 인기 스킬태그 Top N — "평균연봉"처럼 API에 없는 값은 절대 표시하지 않고 실제 제공 지표(공고량·태그 빈도)로 대체 | `GET /tags/categories`, `GET /tags/skills`, `GET /jobs` (category_tags 집계) |
| 7 | 기업 상세 인사이트 | 워치리스트 기업 클릭 시 사업자 정보(설립일, 업력, 고용보험 가입자 수 추이로 회사 성장 규모 추정) | `GET /insight/company` (사업자번호 필요) |
| 8 | 스킬/자격증 준비 현황 | 스킬별 준비도(%), 자격증 취득/준비중 상태 | 사용자 입력 (자체 DB) |
| 9 | 이달의 목표 달성률 | 신규지원 건수, 모의면접 횟수 등 목표 대비 진행률 | 사용자 입력 (자체 DB) |

## 5. 있으면 좋은 기능 (Nice-to-have, 이번 범위 내 업그레이드 후보)

- 공고 상세 모달 (주요업무/자격요건/우대사항 원문 표시) — `GET /jobs/{job_id}`
- 관심기업 신규 공고 알림 배지 (마지막 동기화 이후 새로 열린 포지션 하이라이트)
- 스킬 태그 자동완성 검색으로 내 스킬 등록 시 원티드 실제 스킬태그(`/tags/skills?keyword=`)와 매칭

## 6. 제외 기능 (Out of scope — 명시적으로 뺌)

- **원티드 지원자 개인 이력서/지원 데이터 자동 연동**: 원티드 Open API는 구직자 개인 계정의 지원내역을 제공하지 않음(`/stat/application/summary`, `/recruit-company/*`, `/ats/*`는 채용담당자·ATS 연동사 전용 API이며 client_id/secret으로 인증된 "기업" 주체 기준 데이터). 따라서 개인 지원 현황은 100% 자체 DB 수동 입력으로 관리하고, 이 사실을 UI에도 명시함.
- AI 자소서/이력서 첨삭 기능 (`/ai/pass/text-prediction`, `/ai/apply/text-prediction`) — 범위 밖, 추후 확장 후보로만 기록
- 실제 이력서 파일 업로드/서류 심사, 결제, 알림톡/이메일 발송
- 다중 사용자 협업, 조직/팀 대시보드, 권한 관리
- 모바일 네이티브 앱 (반응형 웹으로 충분)

## 7. 기술 스택 결정

- **Frontend**: 단일 HTML 파일 (바닐라 JS + Chart.js) — 기존 시안 디자인 계승·고도화
- **Backend/DB**: Supabase (Postgres + Auth + Realtime + Edge Functions)
- **외부 API**: Wanted OpenAPI v1 (`openapi.json` 기준). client_id/client_secret은 브라우저에 절대 노출 금지 → Supabase Edge Function이 서버사이드 프록시 역할
- **동기화 방식**: Edge Function이 주기적으로(Cron) 워치리스트 기업의 Wanted 데이터를 조회해 DB에 upsert → 클라이언트는 Supabase Realtime으로 DB 변경분만 구독 (Wanted API를 직접 폴링하지 않음 → API 쿼터 절약 + 진짜 실시간 UX)

## 8. 성공 지표 (참고용)

- 대시보드 진입 후 3초 내 핵심 지표(퍼널, 일정, 워치리스트) 렌더링
- 관심기업 상태 변경이 별도 새로고침 없이 60초 이내 화면 반영
